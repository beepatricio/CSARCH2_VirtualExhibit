## The Stages

### Stage 1 · Fetch

<div class="district-panel">
  <DistrictPlate index={1} total={3} label="Fetch" color="#eb4b3a" client:load />

The CPU pulls the next instruction out of memory using the program
counter, the memory address register, and the memory buffer register.

  <ComponentPanel accent="#eb4b3a" client:load items={[
    { term: "PC", def: "Program Counter - holds the address of the next instruction to run." },
    { term: "MAR", def: "Memory Address Register - holds the address currently being accessed." },
    { term: "MBR", def: "Memory Buffer Register - holds the value just read from or about to be written to memory." },
  ]} />
</div>

### Stage 2 · Decode

<div class="district-panel">
  <DistrictPlate index={2} total={3} label="Decode" color="#3fae5c" client:load />

Once the instruction sits in the instruction register, the decoder
splits it into an opcode and its operands so the control unit knows
what to do next.

  <ComponentPanel accent="#3fae5c" client:load items={[
    { term: "IR", def: "Instruction Register - holds the instruction currently being decoded." },
    { term: "Decoder", def: "Splits the instruction into an opcode (what to do) and operands (what to do it to)." },
  ]} />
</div>

### Stage 3 · Execute

<div class="district-panel">
  <DistrictPlate index={3} total={3} label="Execute" color="#f2a52c" client:load />

The ALU performs the operation and the result lands in its destination
register - here, AX. The cycle then restarts from Fetch.

  <ComponentPanel accent="#f2a52c" client:load items={[
    { term: "ALU", def: "Arithmetic Logic Unit - performs the actual computation or data move." },
    { term: "AX", def: "The destination register that receives the result." },
  ]} />
</div>

</div>
